## 1. Общие сведения

### 1.1. Полное наименование системы

Веб-сервис автоматического сбора научной литературы и формирования двуязычных ИИ-дайджестов по направлениям исследований (**KamGU Research Digest**).

### 1.2. Назначение системы

Система предназначена для исследователей, научных групп и аналитиков, которым необходим быстрый обзор публикаций по выбранной теме, регулярный мониторинг научного направления и сохранение результатов в удобном для работы и передачи виде.

Система **не** является учебной платформой (LMS), порталом абитуриента или системой управления курсами. Основной продукт — обзор и мониторинг научной литературы.

### 1.3. Цели внедрения

1. Сократить время на первичный обзор литературы по теме исследования.
2. Обеспечить регулярный мониторинг выбранных научных направлений с фиксацией изменений.
3. Формировать структурированные двуязычные (RU/EN) обзоры с опорой на открытые научные источники.
4. Хранить историю отчётов и обеспечивать их экспорт и безопасный обмен по ссылке.

### 1.4. Пользователи системы

| Категория | Описание |
|-----------|----------|
| **Зарегистрированный пользователь** | Имеет учётную запись; работает со своими дайджестами, архивом, направлениями и расписаниями. Данные изолированы от других пользователей. |
| **Гость по публичной ссылке** | Просматривает конкретный сохранённый дайджест без входа в систему (после выдачи ссылки владельцем). |

Отдельных ролей «студент / преподаватель / администратор организации» в пользовательском интерфейсе не предусмотрено. Администрирование инфраструктуры (ключи API, почта, планировщик) выполняется на стороне развёртывания сервиса.

## 2. Объект автоматизации и границы системы

### 2.1. Входит в состав системы

- Веб-интерфейс пользователя (браузер).
- Сервер приложения (API) с логикой сбора источников, ранжирования и формирования дайджестов.
- Хранилище данных (база): архив дайджестов, снимки направлений, расписания, учётные записи.
- Встроенный планировщик периодических задач (опционально включаемый).

### 2.2. Внешние зависимости (не разрабатываются в рамках ТЗ)

| Источник / сервис | Назначение |
|-------------------|------------|
| OpenAlex | Основной каталог научных публикаций |
| Semantic Scholar | Дополнительный источник публикаций (опционально) |
| CORE | Дополнительный источник открытого доступа (опционально) |
| Crossref | Обогащение метаданных по DOI |
| Unpaywall | Доступ к полным текстам открытых статей (опционально) |
| Tavily | Поиск веб-сниппетов в режиме «Web» |
| Облачная LLM (OpenAI-совместимый API) | Генерация текстов дайджестов и анализа трендов |
| SMTP | Уведомления о запуске по расписанию (опционально) |
| HTTP webhook | Уведомление внешней системы о статусе запуска (опционально) |

Доступность и лимиты внешних сервисов определяются их владельцами и условиями использования. Система должна корректно отображать предупреждения при частичных сбоях внешних источников.

### 2.3. Не входит в объём (вне рамок текущего продукта)

- Обучение нейросетевых моделей «с нуля».
- Юридическая экспертиза публикаций и проверка плагиата.
- Система электронного документооборота вуза, кадровый учёт, электронная библиотека вуза как таковая.
- Мобильные нативные приложения (iOS/Android); доступ — через браузер.

## 5. Основные сценарии использования

### Сценарий А. Разовый обзор литературы

1. Пользователь входит в систему.
2. На экране «Дайджест» задаёт тему и параметры поиска.
3. Запускает построение дайджеста.
4. Изучает сводки RU/EN, карточки и таблицу публикаций.
5. Экспортирует библиографию и/или сохраняет отчёт в архив.
6. При необходимости выдаёт коллеге публичную ссылку или скачивает Word.

### Сценарий Б. Регулярный мониторинг направления

1. Пользователь создаёт (или выбирает) профиль направления.
2. Выполняет первый снимок (базовая линия).
3. Настраивает расписание (например, еженедельно).
4. В последующие периоды просматривает дельту и дайджест на экране «Мониторинг» / «Тренды».
5. При накоплении снимков запрашивает ИИ-анализ динамики направления.

### Сценарий В. Совместная работа без учётки у получателя

1. Пользователь сохраняет дайджест в архив.
2. Выдаёт публичную ссылку.
3. Получатель открывает ссылку и читает отчёт без регистрации.
4. При необходимости владелец отзывает ссылку.

## 7. Требования к развёртыванию (контур Заказчика)

Минимальный промышленный контур:

- Сервер (VPS/облако) с Docker;
- Домен и TLS (HTTPS);
- PostgreSQL (в составе стека или внешняя управляемая БД);
- Ключи доступа к LLM и при необходимости к Tavily / прочим источникам;
- Корпоративный почтовый ящик для идентификации в OpenAlex (`mailto`) — рекомендуется.

Детальные шаги развёртывания описаны в эксплуатационной документации проекта (`deploy/README.md`) и не дублируются в настоящем ТЗ.

## 8. Критерии приёмки

Система считается принятой при выполнении следующих условий:

1. **Дайджест.** По тестовой теме в режиме «Статьи» формируется двуязычный результат с карточками и таблицей публикаций; экспорт Markdown/BibTeX/RIS доступен.
2. **Архив.** Результат сохраняется, открывается повторно, экспортируется в DOCX; публичная ссылка открывается без входа и отзывается.
3. **Мониторинг.** Создаются два снимка одного профиля за разные периоды; отображается дельта; доступен журнал при использовании расписания.
4. **Тренды.** Профиль отображается в списке; серия снимков строится на графике; при ≥ 2 снимках доступен ИИ-анализ.
5. **Доступ.** Два разных пользователя не видят чужие архивные записи (при включённой авторизации).
6. **Развёртывание.** Сервис доступен по HTTPS на согласованном домене; проверка здоровья API проходит успешно.

## 9. Ограничения и допущения

1. Качество дайджеста зависит от полноты открытых источников и выбранной LLM-модели.
2. Лимиты внешних API (OpenAlex, LLM и др.) могут влиять на время ответа и полноту корпуса.
3. «Популярность» в мониторинге трактуется как изменение цитирований и ранга внутри выборки пользователя, а не как абсолютный мировой рейтинг.
4. Объём внедрения (число пользователей, политика регистрации, BYOK по умолчанию) уточняется при вводе в эксплуатацию у Заказчика.

## Приложение Б. Глоссарий

| Термин | Значение |
|--------|----------|
| **Дайджест** | Структурированный обзор литературы по теме, сформированный системой с помощью LLM |
| **Снимок** | Зафиксированное состояние выборки публикаций по направлению за период (день или месяц) |
| **Профиль / направление** | Именованная настройка мониторинга одного исследовательского поля |
| **Дельта** | Структурированное сравнение текущего снимка с предыдущим |
| **BYOK** | Использование собственного ключа LLM пользователя («bring your own key») |
| **OA** | Open Access — открытый доступ к полному тексту |
